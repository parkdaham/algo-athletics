# 🧾 알고케어 홈화면 PRD

## ✅ 목적

온보딩을 완료한 사용자가 매일 접하게 되는 홈 화면에서, 개인화된 루틴(기존 개념 기준 '맞춤팩')을 시간대별로 재구성해 안내하고, 섭취 실행을 유도함으로써 사용자의 일상 루틴 속에 무마찰 복약을 정착시키는 것이 목적이다.

---

## 👣 기본 플로우

1. 사용자는 앱 접속 시 홈화면에 랜딩
2. 홈화면 상단에는 개인화된 루틴이 요약된 배너로 제공됨 (ex. 이번주는 '회복 맞춤팩'을 제공해 드릴게요)
3. 홈화면에는 맞춤팩을 이루는 구성이 시간대별로 카드 형태로 제공됨 (ex. 아침, 운동 전, 운동 후)
4. 각 카드에는 해당 시간대에 섭취할 영양제 조합이 요약되어 있음
5. 카드를 탭하면 바텀시트로 상세 루틴 정보 확인 가능 (성분별 용량, 설계 이유 등)
6. 사용자는 현재 시간에 맞는 조합을 본인이 선택해 디바이스로 섭취 실행
7. 사용자가 조합 선택 시 복수 선택 즉 한 번에 먹기가 가능.
8. 완료 상태는 UI 상 명확히 피드백 (ex. 체크됨, 색상 변화)
9. 루틴이 모두 완료된 경우 “오늘의 루틴 완료” 상태로 전환

---

## 📦 주요 구성 요소

### 🎯 오늘의 루틴 카드

- 시간대별로 카테고라이징 되어 제공됨.  

| 구성 요소       | 설명                      |
| ----------- | ----------------------- |
| 요약 조합 정보    | 예: 회복 + 근육 강화를 위한 3종 조합 |
| 현재 섭취 가능 여부 | 현재 시간대일 경우 CTA 활성화      |
| 완료 여부 표시    | 이미 먹은 루틴일 경우 완료 표시      |

→ 기본 루틴 + AI가 설계한 부스터가 함께 반영되어 조합이 완성됨

### 📑 루틴 상세 뷰 (카드 탭 시)

- 성분별 구성 (예: BCAA, 마그네슘 등)
- 각각의 목적 / 효능 / 복용 용량
- 설계 이유 요약 문구 (ex. “고강도 훈련 후 회복 강화를 위해 추천된 조합입니다.”)

### ✅ 섭취 실행 로직

- 기본값: 디바이스에 오늘 루틴 제공 → 사용자가 선택 → 섭취
- 옵션: 현재 모든 루틴을 한 번에 섭취하는 기능도 제공 (사용자가 명시적으로 선택 시)
- 섭취 완료 시, 카드 UI 상태 변경 (체크 표시 or 색상 변화)

---

## 📱 모바일 & 디바이스 연동 고려사항

- **모바일에서는 복약 정보와 현황 탐색만 가능하고, 실제 선택 및 섭취는 디바이스에서 수행**

| 상태    | 설명              | 표시 방식                 |
| ----- | --------------- | --------------------- |
| 미섭취   | 아직 섭취 전         | 기본 카드 UI              |

| 전체 완료 | 오늘 섭취 루틴 완료     | 배너 형태로 축하 메시지 + 회고 유도 |
